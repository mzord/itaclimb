# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Initial setup (deps, DB, assets)
mix setup

# Start dev server
mix phx.server
# or with IEx
iex -S mix phx.server

# Database
mix ecto.migrate
mix ecto.reset       # drop + recreate + seed

# Tests (auto-creates/migrates test DB)
mix test
mix test test/path/to/file_test.exs  # single file
mix test test/path/to/file_test.exs:42  # single test by line

# Pre-commit check (compile with warnings-as-errors, format, test)
mix precommit

# Assets
mix assets.build
mix assets.deploy    # minified, for production
```

## Architecture

Phoenix 1.8 + LiveView 1.1 app. PostgreSQL database connects to host `db` (Docker container in dev).

### Contexts

- **`Itaclimb.Blog`** (`lib/itaclimb/blog.ex`) — Post CRUD. `reading_time` is a virtual field populated after every query via `calculate_reading_time/1` (200 wpm). `increment_post_views/1` does an atomic DB update.
- **`Itaclimb.Accounts`** (`lib/itaclimb/accounts.ex`) — User auth generated by `phx.gen.auth` (magic-link/passwordless flow via email). Extended with `username`, `location`, `profile_picture_url` fields.

### Web Layer

- **Router** (`lib/itaclimb_web/router.ex`): `HomeLive` at `/`, `PostController` REST for posts, auth routes under `/users/*`.
- **`HomeLive`** (`lib/itaclimb_web/live/home.ex`): Shows latest 3 posts + hero image via ImageKit.
- **`UserLive.Registration`**: Handles avatar upload — uploads binary to ImageKit API, stores returned URL on user.
- **`ItaclimbWeb.Helpers.ImageHelpers`** (`lib/itaclimb_web/helpers/image_helpers.ex`): Builds ImageKit CDN URLs with transformation params (e.g. `ik_url("image.jpg", w: 200, h: 200)`).

### External Services

- **ImageKit** — image CDN for serving and uploading user photos. Config under `:imagekit` key in `config/config.exs`. URL endpoint, public key, and private key are all stored there.
- **Swoosh** — email delivery (local adapter in dev; preview at `http://localhost:4000/dev/mailbox`).

### Auth Flow

Uses `phx.gen.auth` magic-link pattern: registration sends a login link to email; no password at login. Scope-based auth via `Itaclimb.Accounts.Scope` — `current_scope` assign is available in all LiveViews.
